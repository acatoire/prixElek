# GitHub Copilot Instructions for prixElek Project

Ad a AI assistant for the project, stay KISS (Keep It Simple, Stupid) and follow these guidelines to ensure code
quality, maintainability, and security while adhering to the specific requirements of the prixElek project.
Do not generate too much report files, only generate code files when necessary and requested.
Focus on generating clean, well-documented code that meets the outlined standards.

## Language & Code Standards

**Primary Language:** English (100% for code)

- All variable names, function names, methods, comments in English
- French used only in documentation (doc/ folder)
- Exception: Comments explaining French business context may include French terms in parentheses

**Framework & Tools:**

- Frontend: React or Vue.js (decision in progress) with TypeScript
- Backend: Node.js + Express (if needed)
- Package Manager: npm (Windows PowerShell compatible)
- Code Style: ESLint + Prettier (strict mode)
- Testing: Jest + React Testing Library (or Vue Test Utils)

## Code Architecture

### Adapter Pattern (for Suppliers)

- Base class: `FournisseurAdapter` with abstract methods
- Each supplier: `RexelAdapter`, `SoneParAdapter`, YessAdapter`, `MaterielEltriqueAdapter`
- Location: `src/adapters/`
- Must implement: `authenticate(login, password)` and `getPrice(reference, token)`
- Return types: TypeScript interfaces for type safety

### Module Structure

- Each module in `src/modules/` follows MVC pattern
- Service layer handles business logic
- Components are reusable React/Vue components
- State management via Context API (React) or Pinia (Vue)

### Services Layer

- `PriceService` — orchestrates all adapter calls
- `CatalogService` — loads and manages catalogue.json
- `TokenService` — manages supplier tokens in localStorage
- `CacheService` — implements price caching with TTL

## Security Requirements

### Token Storage

```typescript
// Tokens must NEVER be:
// - Logged to console
// - Stored in version control
// - Sent to external servers
// - Hardcoded in code

// Correct approach:
const tokenStore = {
    save: (supplierId: string, token: string) => {
        localStorage.setItem(`prixelek_token_${supplierId}`, token);
    },
    get: (supplierId: string): string | null => {
        return localStorage.getItem(`prixelek_token_${supplierId}`);
    },
    clear: (supplierId: string) => {
        localStorage.removeItem(`prixelek_token_${supplierId}`);
    }
};
```

### Adapter Error Types

```typescript
interface FetchError {
    code: 'AUTH_ERROR' | 'NETWORK_ERROR' | 'NOT_FOUND' | 'RATE_LIMIT' | 'TIMEOUT';
    supplierId: string;
    message: string;
    statusCode?: number;
    retryable: boolean;
}

// Example usage
if (error.code === 'AUTH_ERROR') {
    // Notify user to reconnect
}
if (error.retryable && retryCount < 3) {
    // Implement exponential backoff
}
```

### Request Limits

- Minimum 500ms between supplier API calls
- Timeouts: 10 seconds per request
- Max 3 retries with exponential backoff (1s, 2s, 4s)
- Rate limit detection: 429 status → back off for 60 seconds

## Testing Requirements

### Coverage: 100% mandatory

Every new file must have a corresponding test file. The CI pipeline will **fail** if coverage drops below 100%.

- **Lines, Branches, Functions, Statements:** all must reach 100%
- Test all adapter methods, services, and utilities
- Mock all external API calls and localStorage
- Co-locate tests: `src/adapters/rexel.ts` → `src/adapters/rexel.test.ts`

### Running Tests (PowerShell)

```powershell
# Run all tests once
npm run test

# Run with coverage report (must show 100%)
npm run test:coverage

# Interactive watch mode during development
npm run test:watch

# Open visual UI in browser
npm run test:ui
```

Coverage report is generated in `coverage/` — open `coverage/index.html` to browse it:

```powershell
# Open coverage report in default browser (Windows)
Start-Process coverage/index.html
```

### Coverage Configuration (vitest.config.ts)

Vitest is configured to **enforce 100%** and fail the build otherwise:

```typescript
// vitest.config.ts
coverage: {

  thresholds: { lines: 100, branches: 100, functions: 100, statements: 100 },
  exclude: ['src/main.ts', '**/*.d.ts', 'vite.config.ts']

```

### Example Test Structure

```typescript
import { describe, it, expect, vi, beforeEach } from 'vitest';
import { RexelAdapter } from './rexel';

describe('RexelAdapter', () => {
    let adapter: RexelAdapter;

    beforeEach(() => {
        adapter = new RexelAdapter();
        vi.clearAllMocks();
    });

    it('authenticate returns token on success', async () => {
        vi.stubGlobal('fetch', vi.fn().mockResolvedValue({
            ok: true,
            json: async () => ({ token: 'abc123' }),
        }));
        const result = await adapter.authenticate('user@test.com', 'pass');
        expect(result).toEqual({ success: true, token: 'abc123' });
    });

    it('getPrice returns price object on success', async () => {
        vi.stubGlobal('fetch', vi.fn().mockResolvedValue({
            ok: true,
            json: async () => ({ price: { net: 4.20 }, salesUnit: 'pièce' }),
        }));
        const result = await adapter.getPrice('LEG-050430', 'abc123');
        expect(result).toEqual({ prix_ht: 4.20, stock: null, unite: 'pièce' });
    });

    it('throws NETWORK_ERROR and retries on failure', async () => {
        vi.stubGlobal('fetch', vi.fn().mockRejectedValue(new Error('Network fail')));
        await expect(adapter.getPrice('REF', 'token')).rejects.toMatchObject({ code: 'NETWORK_ERROR' });
    });
});
```
